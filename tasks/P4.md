# P4 — Soft launch loop
Mostly an operating ritual; Claude Code builds the tooling, the human runs launch/ads.

## P4.1 Economy simulation script
**Steps:** `tools/EconomySim.lua` (pure, runs under test runner): simulate a player
through levels/mastery/prestige/quests over N sessions using real configs; output
coins/gems/XP curves + time-to-unlock tables as CSV/markdown.
**Accept:** runs headless; flags any progression dead zone (>X sessions with no
unlockable goal) per thresholds in the script config.

## P4.2 Remote-tunable balance
**Steps:** Hot-value layer: a `LiveTuning` DataStore/MemoryStore-backed override table
(whitelisted keys only — curve constants, prices, drop weights) merged over config at
boot + on admin command. Guard: schema-validate overrides; never allow new keys.
**Accept:** change a price in dev via override → takes effect without redeploy;
non-whitelisted key rejected + logged.

## P4.3 Weekly funnel ritual (recurring)
**Steps:** Add `docs/WEEKLY_REVIEW.md` template: pull metrics per docs/METRICS.md →
identify top funnel leak → one fix shipped → note result next week. Claude Code's role:
implement the chosen fix as a normal task each cycle.
**Accept:** template committed; first cycle completed and logged.
