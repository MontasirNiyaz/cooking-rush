# P2 — Telemetry

## P2.1 TelemetryService wrapper
**Steps:** Server `TelemetryService` wrapping Roblox AnalyticsService (funnel, economy,
custom events) so call sites are one-liners: `Telemetry.funnel(player, "first_serve")`,
`Telemetry.economy(player, "sink", "upgrade", 250)`. Client events relay through one
rate-limited remote; server stamps truth (no client-supplied amounts).
**Accept:** unit test on the pure event-shaping; rate limit covered; no direct
AnalyticsService calls outside the service.

## P2.2 Instrument funnel + economy + failures
**Steps:** Emit: session_start → first_interact → first_serve → first_level_complete →
(D1 return computed offline); every coin/gem source+sink with reason; level_fail with
{restaurantId, index, coinsAtFail, failReason}; session_length on leave.
**Accept:** grep-able single registry of event names in `Shared/Config/Telemetry.lua`;
manual playthrough produces the full funnel sequence in logs.

## P2.3 Metrics doc
**Steps:** `docs/METRICS.md`: definitions for D1/D7, median session, funnel conversion,
level drop-off curve; where each is read; weekly review checklist (used in P4.3).
**Accept:** doc exists, names match the event registry exactly.
