# P3 — Meta-progression spine
Designs: docs/COOKING_GAME_ROADMAP_M7-M12.md §M7–M8 (authoritative for schemas/formulas).

## P3.1 Recipe Mastery
**Steps:** `Shared/Config/Mastery.lua` per roadmap §7.1; profile field `mastery`;
server-side XP increment on validated serves; `EconomyMath` folds mastery bonus
(pure + tested); mastery bar UI on menu screen.
**Accept:** formula unit-tested; serve N times in dev → level-up persists across rejoin;
no client-writable mastery path.

## P3.2 Restaurant Prestige + hub visibility
**Steps:** `Prestige.lua` curve constants; `EconomyMath.prestigeMultiplier` (pure+tested);
ProgressionService franchise transaction (validate full 3-star completion → reset stars/
upgrades → grant multiplier + prestige tokens, idempotent); hub building gains prestige
flair tier (extend HubPlots variants); prestige badge on avatar nameplate.
**Accept:** franchise round-trip in dev; double-fire of the remote cannot double-grant;
hub visual updates; multiplier visibly affects next level's payout.

## P3.3 Chef collection core
**Steps:** `Chefs.lua` + `RecruitCrates.lua` schemas per roadmap §8; `RecruitService`
(server RNG, weighted rolls, pity counter on profile, published odds endpoint for UI);
inventory + equip slots (cap from prestige level); fusion (N dupes → +1 chef level,
config-driven); crate/inventory/equip UI.
**Accept:** drop-roll + pity logic pure and unit-tested (incl. pity floor exactly at N);
odds shown in UI match config; fusion conserves currency invariants; all grants
server-side and idempotent.

## P3.4 Chefs visible + effective
**Steps:** Equipped chefs follow avatar in hub (pooled models, max 3 visible);
at level start, equipped chefs' passive tags merge into the modifier pipeline
(same mechanism as upgrades — modifiers over base config, never mutation);
implement tag consumers: cookSpeedMult, tipMult, autoServe, burnImmuneChance.
**Accept:** each tag has a unit/integration test proving its effect; unequipping
mid-hub never leaks models; modifier stacking order documented in code comments.

## P3.5 Daily quests + streaks
**Steps:** `Quests.lua` templated objectives + daily roll (seeded per player+date,
pure+tested); QuestService tracks progress server-side from existing telemetry-adjacent
events; login streak with escalating rewards, reset on miss; quest UI panel in hub.
**Accept:** date-rollover test (mock clock); streak math pure-tested; rewards
server-granted only.

## P3.6 Starter cosmetic store
**Steps:** Minimal `Cosmetics.lua` (3–5 decor themes, 3–5 chef skins, name colors);
equip + persistence; small Robux dev-product/gamepass wiring for 1–2 items as the
monetization pipeline proof. Zero gameplay power.
**Accept:** purchase → grant → equip → rejoin persists; receipts processed idempotently
(ProcessReceipt retry-safe).
